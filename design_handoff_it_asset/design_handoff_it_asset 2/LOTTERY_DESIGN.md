# 抽奖功能设计优化(Lottery Redesign)

> 配套原型:`prototype/年会抽奖 · 大屏设计.html`
> 对应现有代码:`frontend/src/pages/Lottery.tsx`(已实现,功能完整但视觉朴素)
>
> 本文档是**纯前端视觉/交互升级**建议 —— 后端 `modules/lottery/`(service/router/models)逻辑扎实,**不需要改动**。

---

## 1. 现状评估

现有 `Lottery.tsx` 是一个标准 Ant Design 表单页:
- 顶部独立 header(不挂 AppLayout sidebar)— ✅ 这个决策对,抽奖适合全屏
- 配置区:活动名称 + 中奖人数 + 关联奖品 SKU + 开始按钮
- 结果区:中奖名单 chips(渐变卡片)
- 历史记录列表

**问题**:抽奖本该是**有仪式感的高光时刻**(年会大屏、全场注视),但现在就是个普通后台表单 —— 点一下按钮,名单"啪"地出现,没有悬念、没有滚动抽取、没有揭晓的爽感。

**后端已有的好设计(保留)**:
- 系统级 RNG(`secrets.SystemRandom`)— 不可预测
- 防重抽(活动名做幂等键,同名拒绝)
- 仅在职 Lark 员工进奖池(排除本地/密码账号)
- 审计日志(`lottery.draw`)

---

## 2. 优化设计:三阶段大屏抽奖

把抽奖从「表单」升级为「**舞台**」,分三个阶段:

### 阶段 1 · idle(待抽)
- 深色星空渐变背景 + 奖项色光晕(特等金 / 一等橙 / 二等蓝 / 三等绿)
- 中央大奖牌 emoji 漂浮动画
- "准备抽取 N 位幸运儿"
- 右侧玻璃拟态配置卡(活动名 / 奖项等级 / 人数步进器 / 奖品)

### 阶段 2 · rolling(滚动抽取)⭐ 签名时刻
- 点击"开始抽奖" → 进入 3.2 秒滚动
- 中央 N 个卡片**高速循环随机姓名**(每 75ms 刷新 + 轻微模糊)
- 营造"老虎机"式悬念
- 按钮变为"正在抽取…"禁用态

### 阶段 3 · reveal(揭晓)⭐ 签名时刻
- 滚动停止 → 最终中奖者**逐个弹出**(`winnerPop` 缩放入场,错峰 0.12s)
- 满屏**彩带 confetti**飘落动画(80 片)
- 每张中奖卡:头像圆 + 大号姓名 + 部门角色 + 奖项 emoji 角标
- "🎉 恭喜中奖 🎉" + 奖品名
- 按钮变"✨ 再抽一轮"

---

## 3. 关键设计决策

### 3.1 奖项等级(新增)
原版只有"中奖人数",新增**奖项等级**(特等/一等/二等/三等),每级一个主题色 + emoji。
整个舞台的光晕、按钮、中奖卡颜色都跟随当前奖项 —— 一等奖一片橙金,二等奖一片蓝,视觉层级清晰。

> ⚠️ 后端 `lottery_draws` 表目前没有 tier 字段。两个选择:
> - **A(推荐)**:加一个 `tier` 字段(varchar,nullable),纯展示用
> - **B**:不加字段,把奖项写进活动名(如"2026年会·一等奖"),前端从名称解析。零后端改动,但弱。

### 3.2 滚动抽取是"表演"不是"真随机过程"
**重要**:滚动阶段的快速姓名是**前端纯视觉效果**,真正的中奖结果由后端 `run_draw` 一次性返回。
前端流程:
1. 点击 → `POST /lottery/draws` 立即拿到真实 winners(后端系统级 RNG)
2. 前端**先播 3.2s 滚动动画**(用随机姓名填充,纯表演)
3. 动画结束 → 展示后端返回的真实 winners

这样**既有仪式感,又不破坏后端的公平性和审计**。原型里为了独立演示用了前端随机,生产对接时改成上面的流程。

### 3.3 玻璃拟态配置卡
右侧配置区用 `rgba(255,255,255,0.05)` + `backdrop-filter: blur(8px)`,深色背景上更精致,符合大屏舞台调性。

### 3.4 防重抽提示前置
原版是点了按钮才报错。新版:活动名一旦和历史记录撞名,**立即**在按钮下方红字提示 + 按钮置灰。减少无效点击。

## 4. 视觉令牌

复用 Phase 1 的字体/间距,但抽奖是**独立深色主题**(不套用浅色后台 token):

```css
/* 舞台背景 */
--lottery-bg: radial-gradient(ellipse at 50% 0%, #1B2A52 0%, #0C1124 55%, #080B18 100%);

/* 奖项色 */
--tier-special: #FFB31A;  /* 特等 金 */
--tier-first:   #FF8800;  /* 一等 橙 */
--tier-second:  #5B92FF;  /* 二等 蓝 */
--tier-third:   #00B42A;  /* 三等 绿 */

/* 玻璃卡 */
--glass-bg: rgba(255,255,255,0.05);
--glass-border: rgba(255,255,255,0.1);
--glass-blur: blur(8px);
```

动画(关键帧见原型 `<style>`):
- `lotteryFloat` — 奖牌漂浮(3s)
- `rollBlur` — 滚动姓名模糊(0.15s alternate)
- `winnerPop` — 中奖卡弹出(0.5s cubic-bezier 错峰)
- `confettiFall` — 彩带飘落(2.4-4.2s)
- `lotteryGlow` — 背景光晕呼吸(4s)

## 5. 实施清单(改造 `Lottery.tsx`)

### P0(核心体验)
1. 三阶段状态机:`useState('idle' | 'rolling' | 'reveal')`
2. 深色舞台布局(左舞台 + 右配置/历史栏)
3. **滚动抽取动画**:`setInterval` 75ms 刷新随机姓名,3.2s 后停
4. **揭晓动画**:`winnerPop` 错峰 + confetti 组件
5. 对接真实接口(见 §3.2 流程):先请求拿结果,再播动画

### P1
6. 奖项等级选择 + 主题色联动(需后端加 `tier` 字段或走方案 B)
7. 防重抽前置校验(活动名撞历史立即提示)
8. 人数步进器替代 InputNumber(大屏更易点)

### P2(锦上添花)
9. 抽奖音效(滚动 tick 声 + 揭晓欢呼)— `<audio>`,可开关
10. 全屏 API(`requestFullscreen`)一键投屏
11. 中奖者头像用真实 Lark 头像(现在用姓名首字占位)
12. "导出中奖名单"按钮(CSV / 复制)

## 6. 接口对接(无需改后端)

现有接口完全够用:
```
GET  /api/lottery/eligible-count    → { count }       奖池人数
GET  /api/lottery/draws             → Draw[]           历史
POST /api/lottery/draws             → Draw             执行抽奖(系统级 RNG)
     body: { name, winner_count, prize_sku_id? }
GET  /api/skus                      → 奖品 SKU 下拉
```

唯一可选的后端改动:`lottery_draws` 加 `tier` 字段(见 §3.1)。
