## 1. 我对当前系统的架构理解

## 1.1 Monorepo 分层
- `backend/`：FastAPI + SQLAlchemy + Alembic + Celery 的业务后端，承担鉴权、资产/库存/审批等核心领域逻辑。
- `frontend/`：React + Vite 的管理端与员工端（`/m`）统一前端。
- `design_handoff_it_asset/`：PRD、设计规范、开发计划与原型基准文档。

## 1.2 后端入口与模块装配
- 应用入口在 `backend/app/main.py`，通过 `include_router` 聚合了 users、assets、inventory、inspections、approvals 等模块路由。
- 启动生命周期中包含 `ensure_initial_admin`，支持首次管理员引导。
- 提供 `/health` 与 `/health/ready`，并将 DB 探活纳入 readiness。

## 1.3 配置与区域化（飞书/Lark）
- `backend/app/config.py` 已将飞书/海外 Lark 端点抽象为 `lark_variant` + 可覆盖 base url，满足“国内/海外双支持”的架构要求。
- 同时存在 JSSDK URL、事件订阅密钥、机器人 chat id 等关键配置项，具备上线前参数化基础。

## 1.4 前端路由结构
- `frontend/src/router.tsx` 体现了管理端（带 `AppLayout`）与员工端（`/m`）并存。
- 鉴权由 `ProtectedRoute` 控制，登录页 `/login` 独立。
