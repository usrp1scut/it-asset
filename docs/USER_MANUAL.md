# 用户手册 · 资产与耗材管理系统

面向 **IT 管理员**与**普通员工**的操作手册。系统分两端:
- **管理端(Web 后台)** —— IT/行政在电脑浏览器使用,负责资产、库存、审批、盘点等全流程。
- **员工端(Lark H5)** —— 员工在 Lark/飞书里打开,查看名下资产、自助申请耗材。
- **移动管理台** —— IT 管理员的手机版后台,主打扫码盘点 / 现场录入。

> 部署、Lark 登录配置等运维内容见 [`DEPLOY.md`](../DEPLOY.md) 与 [`LARK_LOGIN_SETUP.md`](LARK_LOGIN_SETUP.md)。

---

## 1. 登录

打开系统地址后进入登录页,两种方式:

| 方式 | 适用 | 说明 |
|---|---|---|
| **Lark 免登** | 在 Lark/飞书客户端内打开 | 点「使用 Lark 登录」,自动以当前 Lark 账号登录;在 Lark 客户端里通常自动完成 |
| **邮箱 + 密码** | 电脑浏览器 / 未接 Lark | 用管理员分配的邮箱密码登录;首次登录后请尽快改密码 |

- 登录后,**员工**自动进入员工端 `/m`;**管理员**进入 Web 后台工作台。
- 忘记密码 / 账号停用,请联系 IT 管理员。

## 2. 角色与权限

| 角色 | 能做什么 |
|---|---|
| **员工 employee** | 员工端:看名下资产、申请领用耗材、查看领用记录 |
| **主管 manager** | 员工的全部 + 审批下属的申请 |
| **IT 管理员 it_admin** | 几乎全部:资产、库存、审批发放、盘点、维修、报废、离职、用户、移动管理台 |
| **行政/采购 procurement** | 审批 / 发放相关 |
| **财务 finance** | 报废审批等财务相关 |
| **系统管理员 sys_admin** | 全部权限 |

> 后台左侧菜单会按角色显示——带「管理」性质的页面(资产类型、盘点、报废、维修、离职、用户管理、移动管理台等)仅管理员可见。

## 3. 管理端(Web 后台)

左侧菜单分三组:**主要工作 / 流程管理 / 工具与系统**。

### 3.1 工作台
登录后首页。资产/库存/待办的概览看板:资产总数与状态分布、低于安全库存的耗材、待审批、近期动态等。

### 3.2 资产台账(`资产台账`)
全部资产的列表与详情,是日常操作的核心页。

**资产分两类**(由「资产类型」决定):
- **个人发放**(电脑/显示器/手机等):有责任人,走 **分配 → 归还/转移**,离职时回收。
- **基础设施**(网络设备/打印机等):无责任人,只登记台账与位置,直接 **启用 / 停用**。

**资产编号**:`前缀-4 位序号`(如 `PC-0001`、`NET-0003`),系统自动分配、不重复。

**状态**只有四种:在用 / 闲置 / 维修中 / 已报废。

#### 常用操作
- **新增资产**:点「新增资产」,选资产类型(决定编号前缀与个人/基础设施),填品牌型号、配置、序列号(可点图标**扫条形码**录入)、采购日期(日历选择)、采购价等。
- **复制录入**:在某资产详情里点「**复制录入**」,会带着它的类型/型号/配置/地点/供应商/采购信息打开新增表单,**只需改序列号**——批量录入同型号设备很省事(责任人、序列号不会被复制)。
- **导入 / 导出**:支持从云文档/Excel 批量导入(脏数据会先入库并标「待核」,不卡流程);可导出带编号的清单用于贴标或回写。
- **查看 / 编辑**:点任意一行打开右侧详情抽屉(基本信息 / 生命周期 / 配件绑定 / 二维码 / 维修记录 / 附件)。点「编辑」改字段。
  - **改资产类型**:若新类型前缀不同,编号会**重新生成**(如 `PC-0001 → MON-0007`),原实物标签作废、需重打,系统会弹窗确认。
- **分配 / 转移 / 归还**(个人资产):闲置→「分配给员工」;在用→「转移」给他人或「归还入库」。
- **启用 / 停用**(基础设施):闲置→「启用」,在用→「停用」。
- **报修**:开维修工单,资产转「维修中」(详见维修中心)。
- **申请报废**:填原因提交,需**另一位管理员**在「资产报废」页批准后才真正报废(双人复核)。

#### 打印二维码标签
选中一个或多个资产 → 点「打印标签」:
- **选布局**:紧凑 4×8 / 标准 3×6 / 大号 2×4,或 **亚银 A204 模切(66×47mm)**——用对应的物理标签纸时选 A204,列才对得齐。
- **起始位置(复用余纸)**:弹窗里的格子网格,**点哪一格就从哪开始打**,前面已撕掉的位置自动留空——撕剩的标签纸能接着用,不浪费。
- 预览实时刷新,确认后「下载 PDF」打印。
- ⚠️ **打印时必须选「实际大小 / 100%」,不要勾「适合纸张」**,否则二维码会缩放错位。

#### 扫码定位
扫资产二维码可直接跳到该资产(深链 `…/assets?code=PC-0001`)。

### 3.3 资产类型(`资产类型`,管理员)
维护资产分类:名称、编号前缀、个人/基础设施类别、分类图标与颜色。新增类型即可用于新增资产。

### 3.4 库存物品 / 耗材(`库存物品`)
管理耗材与可领用物品:
- **SKU**(物品规格)、分类、单位、安全库存;
- **入库**(收货)、**领用**(发放)、**流水**(每笔出入库记录);
- **低库存预警**:低于安全库存的物品会高亮,并可推 Lark 提醒。

### 3.5 审批中心(`审批中心`)
员工申请 → 主管/IT 审批 → IT 发放 的中枢。

- **待办处理**:卡片显示申请人**真实姓名**、物品**真实名称/规格**;可「通过 / 驳回」,支持填**审批意见**。
- **批量审批**:勾选多条一次性通过/驳回。
- **历史**:切到「全部」可查任意状态的历史申请。
- **IT 发放**:审批通过后,IT 点「发放」——**耗材类自动扣库存**;资产类由 IT 在台账手工分配。
- **自动审批规则**(管理员可配):满足条件(如仅耗材、数量/金额不超上限、且 SKU 未标"需审批")的申请**自动通过**,留「系统自动」标识,直接进 IT 发放,减少人工。
- **Lark 卡片**:申请可推到 Lark,审批人直接在卡片上点[同意]/[拒绝]回写系统。

### 3.6 资产盘点(`资产盘点`,管理员)
- **建任务**:选盘点范围(个人在用 / 所有个人 / 基础设施 / 按地点 / 按部门),系统快照出待核资产清单。
- **扫码核对**(推荐用手机移动管理台):扫资产二维码后,**先弹出该资产详情**(型号/责任人/地点/状态),核对实物一致后**手工点「确认核对一致」**;不一致点「标记差异」并填原因。**不会扫到就直接确认**。
- 进度按 待核 / 已确认 / 差异 统计;盘完「关闭任务」归档。

### 3.7 维修中心(`维修中心`,管理员)
维修工单的全流程:报修开单(内部修 / 外送,记送修日、预计回)→ 维修中 → 完结(记解决说明、费用、是否保修内、新保修期)。完结后资产自动归还/置回闲置。

### 3.8 资产报废(`资产报废`,管理员)
**双人复核**:一名管理员在台账「申请报废」,另一名管理员/财务在此页**审批**并录入处置方式,资产才真正变「已报废」。可看审批链。

### 3.9 离职归还(`离职归还`,管理员)
员工离职时回收其名下资产:
- **建单**:手工建,或由 Lark「员工离职/删除」事件、逾期扫描**自动建单**;
- **重要**:自动化触发时**只通知 IT**,由 IT 核对后点「**确认并通知员工**」才会给员工/上级发 Lark 提醒——避免误触发打扰;
- 逐项**确认归还**(复用归还流程)或**登记丢失**(挂报废核销),全部处理完「关闭」。

### 3.10 用户管理(`用户管理`,管理员)
查看/管理用户与角色;用户身份字段以 Lark 通讯录为准,角色在本系统分配。

### 3.11 操作日志(`操作日志`)
关键操作的审计记录(谁、何时、对什么资源做了什么),可追溯。

## 4. 员工端(Lark H5 · `/m`)

员工在 Lark/飞书里打开,极简三块:
- **我的资产**:名下资产卡片(型号、编号、状态);
- **申请领用耗材/配件**:选物品 → 填事由 → 提交,走「提交人 → 部门主管 → IT 发放」审批流;
- **耗材领用记录**:历史领用(显示物品**真实名称**、数量、日期)。

提交后会通知审批人,员工可在「待办」看到进度。

---

## 5. 移动管理台(`/m/admin`,管理员)

IT 管理员的手机版后台,适合**现场作业**:
- **首页**:待办 / 低库存 / 近期审批快览,九宫格入口;
- **扫码**:扫资产码看详情、就地操作;
- **资产台账**:列表 / 新增 / 详情 / 编辑(全屏移动表单,序列号支持扫码);
- **库存**:库存与预警;
- **审批**:列表 + 底部抽屉处理(显示真实姓名 / SKU 名);
- **盘点**:进任务,**扫码 → 看详情 → 手工确认/标记差异**,支持「连续模式」确认后自动开下一次扫描。

## 6. 常见问题(FAQ)

**Q:打印标签左右两列对不齐?**
用 A204 模切纸时,布局要选「**亚银 A204 模切**」;并确保打印对话框选「**实际大小/100%**」,不要「适合纸张」。仍有微小偏移可联系 IT 调间隙参数。

**Q:发版/部署后看不到新功能?**
浏览器缓存了旧页面。按 **Ctrl + Shift + R** 硬刷一次即可(此后系统已配置自动拿新版,无需再手动刷)。

**Q:页面放久了刷新报 `ERR_EMPTY_RESPONSE`?**
空闲连接被中间网络设备断开导致,**再打开/刷新一次**即可恢复;已在服务端做了缓解。持续频繁出现请反馈 IT。

**Q:Lark 免登报 `invalid url` / 10236?**
属部署配置问题(重定向 URL 未含 `/login` 路径等),见 [`LARK_LOGIN_SETUP.md`](LARK_LOGIN_SETUP.md),由 IT 处理。

**Q:导入的资产显示「待核」?**
导入时责任人/部门没匹配上 Lark 账号或字段缺失,系统先入库并标「待核」,不影响使用;补全信息后可在台账消除标记。
